# MER-013 Meridian App Foundation

## Status
Canon Publication

## Category
Application Architecture

## Purpose

This publication defines the foundational identity of the Meridian application.

Meridian is a personal growth operating system designed to help the user reduce procrastination, maintain momentum, organise daily life, and move towards long-term personal, family, financial, and business goals.

The app must remain simple, premium, useful, calm, and buildable.

## Foundational Statement

Meridian does not measure perfection. It measures momentum.

## Core Product Identity

Meridian is not a noisy productivity app.

Meridian is a calm life-management system that answers one practical question each day:

What is the next right thing to do?

## Primary User

The first version of Meridian is designed for the user personally.

The architecture may later support wider distribution, but it must first work for one person: the user, supported by Ashlar.

## Core Problems

Meridian exists to help with:

- Procrastination
- Loss of focus
- Overcommitment
- Avoidance of difficult conversations
- Daily overwhelm
- Lack of consistent planning
- Losing track of habits, tasks, and long-term goals

## Core Experience

The app should help the user:

- See today's tasks clearly
- Know the next appointment
- Track habits without punishment
- Plan tomorrow before the day begins
- Reflect on lessons learned
- Recover after disruption
- Maintain visible momentum

## Design Ethos

Meridian must feel:

- Minimal
- Premium
- Simple
- Wholesome
- Useful
- Sleek
- Calm

Every interaction must have a purpose.

## Key Architectural Commitments

### Momentum Over Perfection

The app should reward meaningful continuation rather than punish imperfection.

### Recovery Over Shame

Missed days are missed opportunities, not failures.

### Clarity Over Clutter

The app should reduce overwhelm, not create more of it.

### Support Before Scale

The first architecture serves the user directly before considering public distribution.

### Understanding Before Implementation

No major feature should be built before its purpose, behaviour, and user value are understood.

## Core Modules

The Meridian foundation includes the following major modules:

1. Home Screen
2. Habits and routines
3. Tasks and priorities
4. Calendar and planning
5. AI coach
6. Notifications and check-ins
7. Evening review
8. Long-term goals
9. Implementation packs

## Ashlar Role

Ashlar acts as steward of the system.

Ashlar should preserve continuity, identify realistic opportunities, encourage recovery, celebrate meaningful progress, and suggest refinement where appropriate.
