## Meridian Press Findings

### Finding 1: Meridian Press is not yet present in the repository

Repository search and commit history search did not locate a file or commit containing the phrase "Meridian Press" or the MPP publication prefix.

Therefore, Meridian Press currently has the following classification:

Conversation-recovered / repository-unpublished / constitutional placement required.

### Finding 2: Meridian Press exists in recovered architectural memory but not as repository Canon

The recovered architectural memory describes Meridian Press as the constitutional publishing authority or publishing steward of the Meridian Canon.

However, because there is no repository publication and no MER-011 registration for Meridian Press, it must not yet be treated as repository-complete Canon.

### Finding 3: MPP-numbered publications are not registered

No registered MPP publications were found in MER-011.

Previously remembered items such as:

- MPP-100 — Meridian Press Charter
- MPP-101 — Editorial Governance
- MPP-102 to MPP-106 — planned Meridian Press family

should be treated as recovered conversation-memory candidates, not repository-published Canon.

## Register Reconciliation Findings

### Finding 4: MER-011 is the controlling repository index

MER-011 states that a publication is not complete until it is registered, and that it is the controlling index for Canon-approved Meridian and Ashlar publications.

This is consistent with the Repository Constitution, README, and Editorial Process.

### Finding 5: Founder's Migration Map is an authoritative migration specification, not the current repository register

The Founder's Architectural Register Migration Map identifies a broader set of current Canon, legacy/pre-Canon, and planned future Canon publications.

It states that all named documents must receive repository outcomes such as migration, legacy archiving, republication, roadmap tracking, or retirement.

However, it is not itself the same as MER-011.

### Finding 6: Current Canon in the Founder's Register does not equal repository-complete Canon

The Founder's Migration Map lists several documents as current Canon and marked "To migrate", including:

- MER-200 — Experience Domain Philosophy
- MER-201 — Home Screen Architecture
- MER-202 — Experience Domain Implementation Contract
- MER-210 to MER-216 — Planning Domain series
- MER-300 to MER-306 — Task System series
- MER-320 to MER-323 — Habit System series

MER-011 does not yet register those publications.

Therefore, these should be treated as:

Founder's Register current Canon / repository recovery pending / not yet repository-complete.

## Contradictions and Tensions Identified

### Tension 1: Meridian Press versus MER-012 consolidation

GRA-002 Editorial Governance Audit concludes that editorial practice should be consolidated into MER-012 rather than expanded into separate governance publications.

This creates a potential contradiction with the remembered Meridian Press publication family if Meridian Press is treated as a rival or parallel editorial authority.

Resolution required:

Meridian Press must be constitutionally placed as subordinate to, or operating under, MER-012 unless MER-012 is deliberately amended.

### Tension 2: Founder's Register status versus repository status

The Founder's Migration Map identifies documents as current Canon, while MER-011 only registers a smaller set of repository-published documents.

Resolution required:

Use a two-status model:

1. Founder's Register status.
2. Repository completion status.

This prevents loss of the Founder's Register while preserving the MER-011 completion rule.

### Tension 3: Numbering conflict risk around MER-300

The Founder's Migration Map lists MER-300 as Task System Philosophy.

Earlier recovered conversation memory referred to MER-300 as Editorial Constitution.

Repository evidence currently supports MER-300 as Task System Philosophy in the Founder's Migration Map, but not as a registered file in MER-011.

Resolution required:

Do not create or restore MER-300 until the numbering conflict is deliberately resolved.

### Tension 4: Meridian Press naming and numbering family

If Meridian Press uses MPP numbers, the Canon must define how MPP publications relate to MER publications and MER-011.

Resolution required:

Before drafting MPP-100, establish whether:

- MPP documents are registered in MER-011,
- MPP documents have a separate press register,
- or MPP documents are publication-family records subordinate to MER-012.

Until resolved, no Meridian Press document should be treated as Canon-complete.
