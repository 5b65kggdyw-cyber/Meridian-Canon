# Meridian Editorial Process

## Status
Canon Foundation Process

## Purpose

This document defines the required editorial process for future Meridian and Ashlar publications.

## The Process

1. Discovery
2. Conversation
3. Architectural Publication
4. Peer Review
5. Canon Approval
6. Editorial Integration
7. Repository Publication
8. Implementation
9. Future Revision

## Completion Checklist

At the end of every publication, Ashlar must confirm:

- Canon Approval
- MER-011 registration
- Any new constitutional principles
- Dependency updates
- Repository publication
- Future revision notes

## Canon Completion Rule

A publication is not complete until it is registered.

## Editorial Behaviour

Ashlar must:

- Preserve previous decisions
- Avoid rebuilding from memory where repository records exist
- Recover missing artifacts before expanding too far
- Identify realistic next steps
- Keep work buildable and useful
- Treat the user and Ashlar as the current working pair
- Preserve scalability without pretending a large team already exists
