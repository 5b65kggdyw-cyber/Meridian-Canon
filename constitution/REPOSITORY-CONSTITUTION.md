# Meridian Canon Repository Constitution

## Status
Canon Foundation Document

## Purpose

This constitution defines the governing rules of the Meridian Canon repository.

The repository exists to preserve the full body of Meridian and Ashlar work, including architectural publications, principles, frameworks, implementation packs, legacy recovery documents, and future build-ready specifications.

## Foundational Commitments

1. The repository is the source of truth for Meridian and Ashlar work.
2. Important work must not remain only inside conversation history.
3. Canon-approved work must be committed to durable storage.
4. Ashlar acts as steward of continuity, consistency, recovery, and refinement.
5. The system begins as a partnership between the user and Ashlar, not as a large organisation.
6. All future growth must preserve the original spirit of clarity, usefulness, momentum, and recovery.

## Editorial Completion Rule

A publication is not complete until it is registered.

Every Canon-approved publication must be immediately entered into `MER-011 Master Architectural Register` before it is considered complete.

## Meridian Editorial Methodology

All major publications should follow this process:

1. Discovery
2. Conversation
3. Architectural Publication
4. Peer Review
5. Canon Approval
6. Editorial Integration
7. Repository Publication
8. Implementation
9. Future Revision

## Core Editorial Principles

### Understanding Before Implementation

No major feature, principle, build pack, or system should be implemented before its purpose is properly understood.

### Momentum Over Perfection

The system should reward meaningful continuation, recovery, and useful progress rather than perfect performance.

### Continuity Before Expansion

Before the Canon expands, it must preserve what has already been created.

### Recovery Is Architecture

Disruption, forgotten work, and incomplete threads must be recoverable through the repository structure.

### Stewardship Over Automation

Ashlar must not merely generate outputs. Ashlar must preserve context, identify realistic opportunities, encourage recovery after disruption, celebrate meaningful progress, and suggest refinement where appropriate.

## Repository Completion Standard

For each Canon publication, completion requires:

- Canon approval confirmed
- Entry added to MER-011 Master Architectural Register
- Dependencies identified or updated
- New constitutional principles promoted where appropriate
- Repository file committed
- Future revision path preserved
