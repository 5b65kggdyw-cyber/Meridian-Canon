# Setting Sun Migration Programme

## Status
Active Migration Programme

## Purpose

The Setting Sun Migration Programme exists to recover, refine, register, and preserve the full body of Meridian and Ashlar work inside the Meridian Canon repository.

This programme prevents important architecture, publications, principles, decisions, and implementation work from remaining only inside conversation history.

## Governing Principle

Continuity before expansion.

Before new major work is created, existing work should be recovered into the repository wherever possible.

## Migration Rule

No recovered artifact becomes Canon-complete until it is:

1. Identified
2. Recovered
3. Reviewed
4. Refined using Meridian editorial standards
5. Assigned a MER ID where appropriate
6. Registered in MER-011
7. Committed to the repository

## Migration Phases

### Phase 1: Repository Foundation

Status: Complete

Includes constitution, master register, editorial process, and foundational directories.

### Phase 2: Legacy Census

Status: Active

Identify all known documents, frameworks, principles, app architecture decisions, build packs, and unfinished publications.

### Phase 3: Recovery Queue

Status: Active

Move identified artifacts into a controlled queue for review and refinement.

### Phase 4: Canon Refinement

Status: Pending

Rewrite recovered work using Meridian editorial standards without losing original decisions.

### Phase 5: Registration

Status: Pending

Register each approved publication in MER-011.

### Phase 6: Implementation Readiness

Status: Pending

Convert approved architecture into build-ready implementation packs for Cursor or equivalent tools.

## Immediate Priority

The next task is to create the Legacy Census and Recovery Queue.

The migration should begin with broad recovery before attempting final polish.
